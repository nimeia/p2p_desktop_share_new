## 2. 本应用的核心功能

本应用用于在同一本地网络或热点环境中共享屏幕内容。当前版本主要提供以下功能：

- 在 Windows 桌面端启动本地共享服务
- 通过浏览器或可选嵌入式 WebView2 打开 Host 页面并发起屏幕共享
- 通过浏览器 Viewer 页面在本地网络中观看共享画面
- 导出本地分享与诊断文件，帮助用户在同一局域网内完成连接和排障

本应用当前不提供公共互联网中转服务，不要求用户创建云端账号，也不以内置广告、用户画像或行为追踪为目的处理数据。

## 3. 我们会处理哪些信息

根据你实际使用的功能，本应用可能会访问、处理、传输或在本地保存以下信息：

### 3.1 屏幕共享内容

当你主动点击开始共享并授权浏览器或 WebView2 调用屏幕捕获能力后，本应用会处理你选择共享的屏幕、窗口或标签页内容。

- 该处理仅在你主动发起共享并授予权限后发生
- 屏幕内容用于向你邀请的 Viewer 设备传输和显示
- 本应用当前版本不会将屏幕内容上传到开发者运营的云端服务器

请注意：你共享的屏幕内容可能包含个人信息、工作信息、文件内容、通知内容或其他敏感信息。你应自行决定共享范围，并避免在不受信任的环境中共享敏感内容。

### 3.2 会话与连接信息

为了建立和维持共享会话，本应用会处理以下会话或网络信息：

- 房间号（room）
- 会话令牌（token）
- 主机本地网络地址（例如局域网 IPv4 地址）
- 端口号
- Viewer URL / Host URL
- Viewer 连接数量、房间状态、连接状态
- WebRTC 会话协商相关信息，例如 offer、answer、ICE candidate

这些信息用于建立本地会话、把共享画面发送到 Viewer，以及显示当前连接状态。

### 3.3 诊断与导出文件

当你使用“导出分享信息”“导出诊断信息”“运行自检”或类似功能时，本应用可能在你的设备本地生成文件，例如：

- `share_bundle.json`
- `share_diagnostics.txt`
- `viewer_url.txt`
- `desktop_self_check.txt`
- `desktop_self_check.html`
- `share_card.html`
- `share_wizard.html`
- `share_status.js`
- `hotspot_credentials.txt`

这些文件可能包含以下内容：

- 房间号、会话令牌、Viewer URL、Host URL
- 主机本地 IP、端口、连接状态、Viewer 数量
- 局域网连通性、端口监听状态、防火墙或网络诊断结果
- 当你使用热点功能时，热点 SSID 和密码

这些文件默认用于本地排障、会话交接和离线查看，不会自动上传到开发者服务器。请谨慎分享这些文件，因为它们可能包含可用于连接当前会话的信息。

### 3.4 本地偏好与本地缓存

为了改善本地使用体验，本应用的 Host / Viewer 页面可能会在本地浏览器或 WebView 运行环境中存储有限的本地数据，例如：

- 语言或界面偏好设置
- 为安装式或全屏式页面体验准备的应用外壳缓存

这些信息仅保存在你的设备本地，用于页面加载和界面体验，不用于广告追踪。

### 3.5 日志与状态信息

本应用可能在界面中显示或在本地诊断导出中记录以下信息：

- 运行状态
- 错误信息
- 局域网连接与健康检查结果
- WebView2 运行状态
- 共享过程中的本地调试日志

这些信息用于帮助你识别连接、共享或环境配置问题。

## 4. 我们如何使用这些信息

我们仅将上述信息用于以下目的：

- 建立、维持和结束本地屏幕共享会话
- 在 Host 与 Viewer 之间传输你主动共享的屏幕内容
- 显示连接状态、房间状态和 Viewer 数量
- 生成 Viewer 链接、分享页面和本地诊断结果
- 帮助你排查端口、防火墙、局域网连通性或运行环境问题
- 记住有限的本地界面偏好

## 5. 信息如何共享

除下列情况外，我们不会将你的信息出售、出租或出于广告目的共享给第三方：

- 你主动邀请加入会话的 Viewer 设备会接收你选择共享的屏幕内容以及建立连接所需的本地会话信息
- 你主动导出并分享诊断文件、Viewer URL、房间号、令牌或热点信息时，这些接收方可访问相应内容
- 法律法规、司法程序或监管要求另有规定时

当前版本中，我们不运营用于接收你屏幕内容或诊断文件的开发者云端服务。

## 6. 信息存储与保留

### 6.1 实时会话数据

屏幕共享内容和实时连接状态主要在本地会话期间临时处理，用于让 Viewer 观看共享画面。会话结束后，这类实时传输数据通常不会由本应用作为持久化历史内容保存。

### 6.2 本地导出与本地文件

当你主动执行导出、自检、分享或诊断操作时，相关文件会保存在你的设备本地。保留时间取决于：

- 你是否继续保留这些文件
- 你是否手动删除这些文件
- 操作系统或应用后续版本如何管理运行目录

如果这些文件包含房间号、令牌、Viewer URL 或热点密码，请在不再需要时及时删除。

### 6.3 本地浏览器/WebView 数据

浏览器或 WebView2 运行环境可能保留有限的本地缓存、服务工作线程缓存或界面偏好设置。你通常可以通过浏览器或系统设置清除这些数据。

## 7. 安全说明

我们采取合理措施减少不必要的数据处理，并尽量将数据处理限制在你的本地设备和你选择加入的本地网络参与者之间。

但你也应知悉：

- 当前版本的产品定位是本地网络共享，而不是公共互联网服务
- 当前版本的本地服务与信令通道使用本地网络 HTTP / WS，适合受信任的局域网或热点环境
- 数据安全还取决于你的本地网络安全状况、浏览器/WebView2 运行环境、设备安全配置以及你选择共享的对象

因此，我们建议你：

- 仅在你信任的本地网络或热点环境中使用本应用
- 谨慎分享包含房间号、令牌、Viewer URL 或热点密码的文件
- 避免共享包含敏感信息的屏幕内容，除非你确认接收方和网络环境可信

## 8. 你的控制与选择

你可以通过以下方式控制本应用对信息的处理：

- 仅在你愿意时启动共享，并可随时停止共享
- 选择具体共享哪一个屏幕、窗口或标签页
- 不导出或不分享诊断与分享文件
- 手动删除导出的本地文件
- 清除浏览器或 WebView 的本地缓存与偏好
- 不在不可信网络中使用本应用

如果你使用复制功能，某些信息（例如 Viewer URL、令牌）会在你的主动操作下复制到系统剪贴板。

- Microsoft 隐私声明

- WebView2 相关产品文档
